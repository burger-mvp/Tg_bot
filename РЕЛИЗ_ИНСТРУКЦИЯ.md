## Перед запуском

### 1. Применить миграцию БД
```bash
psql -U your_user -d your_database -f migrations/003_add_shop_and_trusted_sellers.sql
```

Миграция добавит:
- Поле `shop_name TEXT` в таблицу `users`
- Sequence `shop_counter` для автоинкремента
- Роль `'trusted_seller'` в CHECK constraint
- Языки `'fa'`, `'ur'`, `'hi'`, `'bn'` в CHECK constraint
- Статус `'rejected'` в post_queue
- Тип `'photo'` в комментарий media_file_ids

### 2. Обновить существующих пользователей (опционально)
Если нужно присвоить shop_name уже зарегистрированным пользователям:
```sql
UPDATE users
SET shop_name = 'Shop ' || nextval('shop_counter')
WHERE shop_name IS NULL AND registered_at IS NOT NULL;
```

### 3. Перезапустить бота
```bash
# Docker
docker-compose down
docker-compose up -d --build

# Или локально
python bot.py
```

## Проверка работы

### 1. Тест ReplyKeyboard
- Создайте пост как обычный пользователь
- Убедитесь, что кнопка "Медиа загружены" появляется внизу экрана (не inline)

### 2. Тест фотографий
- Отправьте фото при создании поста
- Убедитесь, что оно сохраняется и публикуется корректно

### 3. Тест новых языков
- Выберите язык дари/урду/хинди/бенгальский при регистрации
- Проверьте что все кнопки и сообщения отображаются на выбранном языке

### 4. Тест округления цен
- Введите цену 1234 AED
- Должна округлиться до 1240 AED

### 5. Тест Shop ID
- Зарегистрируйте нового пользователя
- Проверьте что ему присвоен shop_name (будет виден в опубликованном посте)

### 6. Тест trusted_seller
- Супер-админ: откройте супер-админ панель → "Назначить доверенного"
- Введите имя магазина (например "Shop 5")
- Зайдите под этим пользователем и создайте пост
- Пост должен публиковаться сразу без модерации

### 7. Тест кнопки "Отклонить"
- Обычный пользователь создаёт пост
- Супер-админ видит три кнопки: Одобрить / Редактировать / **Отклонить**
- Нажмите "Отклонить" — автор получит уведомление

### 8. Тест команды /info
- Отправьте `/info` боту
- Должно прийти описание бота на вашем языке

## Известные ограничения

1. **Shop name нельзя изменить** — присваивается один раз при регистрации
2. **trusted_seller может назначить только супер-админ** — обычные админы не имеют доступа
3. **Отклонённые посты не удаляются из БД** — остаются со статусом `'rejected'` для аудита
4. **Фото в документах не поддерживаются** — только video, photo, document (видео-файлы)
