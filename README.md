# Telegram-бот: первая часть

Бот реализован на `aiogram 3.x`, использует PostgreSQL через `asyncpg` и готов к запуску локально или в Docker/Railway.

## Первый запуск локально

1. Создайте и активируйте виртуальное окружение Python 3.10+.
2. Установите зависимости:
   ```bash
   pip install -r requirements.txt
   ```
3. Откройте `.env` и замените все шаблонные значения настоящими данными.
4. Выполните SQL из `structure.sql` в вашей PostgreSQL базе данных.
5. Запустите бота:
   ```bash
   python bot.py
   ```

## Роли

- `super_admin` — пользователь с `SUPER_ADMIN_ID`;
- `admin` — пользователь, чей ID есть в `ADMIN_IDS`;
- `user` — любой другой пользователь.

При первой команде `/start` бот запрашивает номер телефона, сохраняет пользователя и пытается уведомить всех администраторов. Чтобы Telegram позволил отправить уведомление, каждый администратор должен хотя бы один раз открыть чат с ботом и отправить `/start`.

## Railway и Docker

Передайте значения из `.env` как переменные окружения сервиса Railway. Файл `.env` намеренно исключен из Docker-образа и Git. База должна быть создана заранее выполнением `structure.sql`.
