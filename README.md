# Telegram-бот

Бот реализован на `aiogram 3.x`, использует PostgreSQL через `asyncpg` и готов к запуску локально или в Docker/Railway.

## Возможности

- регистрация: выбор языка интерфейса (русский, английский или арабский), затем передача собственного номера через кнопку контакта;
- роли `user`, `admin` и `super_admin` из переменных окружения;
- уведомление всех администраторов о первой завершенной регистрации пользователя;
- FSM-создание поста с 1–30 видео (включая альбомы), выбором категории, расчетом AED → USD и обязательным текстом доставки;
- модерация постов обычных пользователей супер-администратором;
- PostgreSQL-очередь: один пост в 30 минут с 09:00 до 22:00 и один повтор через 7 дней;
- APScheduler, который безопасно восстанавливает задачи повторов после перезапуска Railway.

## Первый запуск локально

1. Создайте и активируйте виртуальное окружение Python 3.11+.
2. Установите зависимости:
   ```bash
   pip install -r requirements.txt
   ```
3. Откройте `.env` и замените все шаблонные значения настоящими данными.
4. Для новой базы выполните SQL из `structure.sql` в PostgreSQL.
5. Для уже существующей базы сначала выполните `migrations/001_add_user_language.sql`, затем `migrations/002_add_post_queue.sql`.
6. Запустите бота:
   ```bash
   python bot.py
   ```

## Обновление существующей Railway-базы

Перед деплоем версии с языками выполните в SQL-консоли PostgreSQL Railway содержимое файла:

```text
migrations/001_add_user_language.sql
```

Миграция `001` добавляет `users.language_code`. Затем обязательно выполните:

```text
migrations/002_add_post_queue.sql
```

Вторая миграция создает таблицу `post_queue`, добавляет метку первой регистрации и не удаляет существующие данные. Она безопасна для повторного запуска.

## Роли

- `super_admin` — пользователь с `SUPER_ADMIN_ID`;
- `admin` — пользователь, чей ID есть в `ADMIN_IDS`;
- `user` — любой другой пользователь.

При первом `/start` бот предлагает выбрать язык. После нажатия выбор сохраняется в `users.language_code`, затем бот запрашивает собственный номер через Reply-кнопку Telegram и сохраняет его в `users.phone_number`. Главное меню отображается только после завершения обоих шагов. В этот момент всем ID из `ADMIN_IDS` и `SUPER_ADMIN_ID` отправляется уведомление о новом пользователе.

Все зарегистрированные роли используют **Создать пост**: загружают от 1 до 30 видео, выбирают товар и вводят цену/описание. Для ДВС применяется наценка 10%, для кузовного — 15%. Обычный пользователь отправляет заявку супер-администратору на модерацию; администратор и супер-администратор ставят её в очередь сразу.

Очередь проверяется в `:00` и `:30` строго с 09:00 до 22:00 включительно. Часовой пояс задается переменной `SCHEDULER_TIMEZONE` и по умолчанию равен `Europe/Moscow` (московское время). После первой успешной публикации пост автоматически повторяется через 7 дней и затем удаляется из `post_queue`.

## Railway и Docker

Передайте значения из `.env` как переменные окружения сервиса Railway. Файл `.env` намеренно исключен из Docker-образа и Git.

Для публикации добавьте бота в целевой канал как администратора с правом **«Публикация сообщений»**. В `CHANNEL_ID` укажите числовой ID канала, обычно начинающийся с `-100`.

Дополнительно можно задать:

```env
SCHEDULER_TIMEZONE=Europe/Moscow
```
