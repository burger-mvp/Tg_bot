# Telegram-бот

Бот реализован на `aiogram 3.x`, использует PostgreSQL через `asyncpg` и готов к запуску локально или в Docker/Railway.

## Возможности

- выбор языка интерфейса: русский, английский или арабский;
- роли `user`, `admin` и `super_admin` из переменных окружения;
- административная FSM-форма создания поста: текст → одно фото (опционально) → предпросмотр → публикация;
- публикация подтвержденных постов в Telegram-канал и сохранение их истории в PostgreSQL.

## Первый запуск локально

1. Создайте и активируйте виртуальное окружение Python 3.10+.
2. Установите зависимости:
   ```bash
   pip install -r requirements.txt
   ```
3. Откройте `.env` и замените все шаблонные значения настоящими данными.
4. Для новой базы выполните SQL из `structure.sql` в PostgreSQL.
5. Для уже существующей базы выполните миграцию `migrations/001_add_user_language.sql`.
6. Запустите бота:
   ```bash
   python bot.py
   ```

## Обновление существующей Railway-базы

Перед деплоем версии с языками выполните в SQL-консоли PostgreSQL Railway содержимое файла:

```text
migrations/001_add_user_language.sql
```

Миграция добавляет `users.language_code`. Она безопасна для повторного запуска. Уже зарегистрированные пользователи при следующем `/start` увидят выбор языка; их существующие данные не удаляются.

## Роли

- `super_admin` — пользователь с `SUPER_ADMIN_ID`;
- `admin` — пользователь, чей ID есть в `ADMIN_IDS`;
- `user` — любой другой пользователь.

При первом `/start` бот предлагает выбрать язык. После нажатия на кнопку выбор сохраняется в `users.language_code`, а затем отображается главное меню на выбранном языке.

Администратор открывает **Админ-панель** и проходит сценарий создания поста: ввод текста, одно фото или пропуск этого шага, предпросмотр и подтверждение публикации.

## Railway и Docker

Передайте значения из `.env` как переменные окружения сервиса Railway. Файл `.env` намеренно исключен из Docker-образа и Git.

Для публикации добавьте бота в целевой канал как администратора с правом **«Публикация сообщений»**. В `CHANNEL_ID` укажите числовой ID канала, обычно начинающийся с `-100`.
