# Changelog: Добавление возможности назначения администраторов

## Дата: 2026-07-14

## Описание изменений

Добавлена возможность для супер-администратора назначать роль "admin" другим пользователям через интерфейс бота. Обычные администраторы теперь могут модерировать, редактировать и одобрять посты наравне с супер-администратором.

## Изменённые файлы

### 1. `loc_data.py`
**Добавлены новые тексты локализации:**
- `set_admin` - текст кнопки "Назначить администратора"
- `enter_telegram_id_for_admin` - запрос Telegram ID для назначения админа
- `admin_assigned` - подтверждение успешного назначения
- `user_not_found` - сообщение об отсутствии пользователя в БД

Локализация добавлена для русского и английского языков.

### 3. `handlers/admin.py`

- Message-обработчик `process_telegram_id_for_admin()` для обработки введённого Telegram ID

**Логика назначения:**

2. Вводит Telegram ID пользователя

## Архитектурные решения

### Разделение ролей
- **Супер-админ**: всегда определяется из `config.SUPER_ADMIN_ID`, не хранится в БД
- **Обычные админы**: назначаются через бот, хранятся в БД в поле `users.role`
- **Fallback**: сохранена совместимость с `ADMIN_IDS` из .env для старых админов

### Права доступа
- **Модерация**: супер-админ + обычные админы (через `is_admin_or_higher()`)
- **Назначение ролей**: только супер-админ (через `is_super_admin()`)
- **Trusted sellers**: не модерируют, только автопубликация без модерации

### Синхронные vs Async проверки
- `get_role()` - синхронная, читает только из config (legacy)
- `get_role_from_db_or_config()` - async, читает из БД с fallback на config
- `is_admin_or_higher()` - async, для проверки прав модерации
- `is_super_admin()` - синхронная, проверяет только супер-админа

## Требования к БД
Роль "admin" уже поддерживается в `structure.sql` и миграции `003_add_shop_and_trusted_sellers.sql`:
```sql
role TEXT NOT NULL DEFAULT 'user' CHECK (role IN ('user', 'admin', 'super_admin', 'trusted_seller'))
```

Дополнительные миграции не требуются.

## Тестирование

### Ручное тестирование
1. Супер-админ открывает супер-админ панель
2. Нажимает "Назначить администратора"
3. Вводит Telegram ID зарегистрированного пользователя
4. Проверяет подтверждение назначения
5. Назначенный админ пробует модерировать пост (approve/edit/reject)

### Проверка падения
- Ввод несуществующего Telegram ID → сообщение "user_not_found"
- Ввод нечислового значения → сообщение "invalid_price"
- Ввод ID незарегистрированного пользователя → сообщение "user_not_found"

## Безопасность
- Назначать админов может только супер-администратор
- Обычные админы не могут назначать других админов
- Роль супер-админа не может быть изменена через бот (только через config)

## Совместимость
- Старые админы из `ADMIN_IDS` в .env продолжают работать
- При первом /start их роль сохраняется в БД
- Новая система приоритетнее: роль из БД переопределяет .env
