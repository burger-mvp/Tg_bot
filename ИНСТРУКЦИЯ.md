# Инструкция по запуску и развертыванию

## 1. Что уже подготовлено

- Python-бот на `aiogram 3.x`;
- асинхронное подключение к PostgreSQL через `asyncpg`;
- регистрация по `/start`: выбор языка, затем ввод номера телефона;
- роли `user`, `admin` и `super_admin`;
- создание и публикация постов администраторами;
- Dockerfile для запуска на Railway;
- виртуальное окружение `.venv` с установленными зависимостями.

## 2. Настройка `.env`

Откройте файл `.env` в корне проекта и замените шаблонные значения:

```env
TOKEN=123456789:AAExampleBotToken
DATABASE_URL=postgresql://postgres:пароль@хост:5432/название_базы
SUPER_ADMIN_ID=123456789
ADMIN_IDS=987654321,111222333,444555666
CHANNEL_ID=-100123456789
```

### Где взять значения

- `TOKEN` — создайте бота через [@BotFather](https://t.me/BotFather), затем выполните `/newbot`.
- `SUPER_ADMIN_ID`, `ADMIN_IDS` — числовые Telegram ID соответствующих пользователей. Бот может узнать свой ID только из входящего сообщения, поэтому каждый администратор должен сначала отправить ему `/start`.
- `CHANNEL_ID` — числовой ID Telegram-канала, обычно начинающийся с `-100`. Бот должен быть добавлен в этот канал администратором с правом публикации сообщений.
- `DATABASE_URL` — строка подключения к PostgreSQL. Ее предоставит Railway после создания базы данных.

> Не публикуйте `.env` и токен бота. Этот файл уже добавлен в `.gitignore` и `.dockerignore`.

## 3. Создание таблиц PostgreSQL

Для новой базы таблицы необходимо создать вручную.

1. Откройте консоль SQL вашей PostgreSQL базы (например, вкладку **Data** в Railway).
2. Откройте файл `structure.sql` в проекте.
3. Скопируйте и выполните его содержимое целиком.
4. Убедитесь, что появились таблицы `users` и `posts`.

Повторное выполнение безопасно: в скрипте используется `CREATE TABLE IF NOT EXISTS`.

### Обновление существующей базы: добавление языка

Перед развертыванием версии с выбором языка выполните в SQL-консоли PostgreSQL Railway файл:

```text
migrations/001_add_user_language.sql
```

Его SQL-код:

```sql
ALTER TABLE users
    ADD COLUMN IF NOT EXISTS language_code VARCHAR(2);
```

Полная миграция также добавляет ограничение, разрешающее только коды `ru`, `en`, `ar` или `NULL`. Она безопасна при повторном запуске. Для старых пользователей поле останется пустым, поэтому при следующем `/start` бот предложит им выбрать язык.

## 4. Локальный запуск в VS Code (Windows)

Откройте встроенный терминал VS Code в папке проекта и активируйте созданное окружение:

```cmd
.venv\Scripts\activate
```

После активации перед приглашением терминала появится `(.venv)`. Запустите бота:

```cmd
python bot.py
```

Остановка бота: нажмите `Ctrl+C` в терминале.

Если окружение еще не создано или зависимости необходимо переустановить:

```cmd
python -m venv .venv
.venv\Scripts\activate
python -m pip install -r requirements.txt
```

### Выбор интерпретатора в VS Code

1. Нажмите `Ctrl+Shift+P`.
2. Выберите **Python: Select Interpreter**.
3. Выберите интерпретатор из `.venv`, обычно `.venv\Scripts\python.exe`.

## 5. Проверка работы в Telegram

1. Откройте бота с нового пользовательского аккаунта и отправьте `/start`.
2. Нажмите одну из кнопок выбора языка.
3. Введите номер телефона обычным текстовым сообщением.
4. Бот сохранит `language_code` и `phone_number`, назначит роль по настройкам окружения и покажет локализованное меню.
5. Откройте бота с аккаунта из `SUPER_ADMIN_ID` или `ADMIN_IDS`.
6. Нажмите **Админ-панель**, введите текст, отправьте одно фото или нажмите пропуск, проверьте предпросмотр и подтвердите публикацию.

Если публикация не отправляется в канал, проверьте, что бот добавлен туда администратором и обладает правом публикации сообщений.

## 6. Развертывание на Railway

### База данных

1. Создайте в Railway проект.
2. Добавьте сервис **PostgreSQL**.
3. Выполните `structure.sql` в SQL-консоли этой базы.
4. Скопируйте внешний или внутренний URL подключения PostgreSQL в переменную `DATABASE_URL` сервиса бота.

### Сервис бота

1. Загрузите проект в GitHub или разверните его способом, который поддерживает Railway.
2. Создайте сервис из этого репозитория. Railway автоматически использует `Dockerfile` из корня проекта.
3. В разделе **Variables** добавьте:

   ```text
   TOKEN
   DATABASE_URL
   SUPER_ADMIN_ID
   ADMIN_IDS
   CHANNEL_ID
   ```

4. Вставьте реальные значения без кавычек и сохраните их.
5. До деплоя выполните `migrations/001_add_user_language.sql` в базе PostgreSQL Railway.
6. Запустите deployment и откройте логи: после успешного старта появятся сообщения `aiogram` о polling.

Файл `.env` на Railway не загружается и не нужен: значения берутся из переменных окружения сервиса.

## 7. Диагностика распространенных ошибок

| Ошибка | Что проверить |
| --- | --- |
| `Не задана обязательная переменная окружения` | В `.env` отсутствует указанная переменная или в Railway она не добавлена в Variables. |
| Ошибка подключения к PostgreSQL | Проверьте `DATABASE_URL`, доступность базы и наличие таблиц из `structure.sql`. |
| При старте ошибка `column language_code does not exist` | Выполните `migrations/001_add_user_language.sql` в SQL-консоли Railway и перезапустите сервис. |
| Не публикуется пост в канал | Добавьте бота в канал как администратора, дайте ему право публиковать сообщения и проверьте `CHANNEL_ID`. |
| `Unauthorized` от Telegram | `TOKEN` неверный, отозван в BotFather или содержит лишние пробелы. |
| Бот не запускается локально | Активируйте `.venv` и выполните `python -m pip install -r requirements.txt`. |
